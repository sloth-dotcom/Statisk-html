Section 4

The SendGrid build

How the strategy actually gets wired up in SendGrid — Marketing Automations for time-based flows, the Mail Send API for real-time events, suppression groups, deliverability and a live mock endpoint you can call below.

Time-based flows
Marketing Campaigns
Contacts, Segments, Automations and Single Sends. Runs onboarding, the trial countdown, renewal reminders, win-back, sunset and the weekly newsletter.
Real-time, event-triggered flows
Email API (Mail Send)
Sends a Dynamic Template by ID on demand. Runs dunning, credit balance, abandonment, milestones and cross-sell — your app calls it the moment the event fires.

Building an automation

  1. Create the segment that defines who enters the flow (data model — Section 2.2).
  2. In Marketing → Automations, create an automation with entry trigger = segment membership.
  3. Add each email as a step, with a delay before it (wait 1 day, wait 3 days, etc.).
  4. Attach the Dynamic Template to each step and set the ASM suppression group.
  5. Design segments that automatically “release” converted contacts — that's the exit.
  6. Send test contacts through every branch before activating.

Event-triggered send — sample request

Real-time flows don't wait on a schedule — your app calls the API the moment the event fires.

POST https://api.sendgrid.com/v3/mail/send
Authorization: Bearer ${SENDGRID_API_KEY}
Content-Type: application/json

Deliverability foundations

Authenticate everything
Domain auth (DKIM CNAMEs), link branding, SPF and DMARC. Start DMARC at p=none to monitor; move to quarantine/reject once clean.
Dedicated marketing subdomain
Send marketing from mail.[brand].com, separate from transactional and corporate. Reputation issues stay contained.
Meet bulk-sender rules
Gmail and Yahoo require auth, one-click unsubscribe (List-Unsubscribe), and spam-complaint rate < 0.3% — aim well below 0.1%.
Warm up a dedicated IP
Shared IPs are fine under ~100k sends/month. On dedicated, ramp gradually: 50 → 100 → 500 → 1k → 5k → 10k → double daily.
Send to the engaged
Frequency cap: 1 marketing email per 24h, 3 per 7 days; billing exempt. Run Flow 3C sunset to retire 180-day non-openers.
Stream the data back
Event Webhook feeds opens, clicks, bounces, spam reports and unsubs into your BI tool for the KPI dashboard and engagement scoring.
